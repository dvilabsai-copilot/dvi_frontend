# Test data and environment

## Environment

- Frontend base URL: `.env.e2e` `E2E_BASE_URL` (currently local Vite on port 8080).
- API base URL: `.env.e2e` `E2E_API_BASE_URL` (currently local API on port 4006 under `/api/v1`).
- Existing E2E helper logs in through `/auth/login`, then sets `accessToken` in browser local storage. Credentials are read from environment variables by the helper; secrets are intentionally not copied here.
- The checked-in `.env.e2e` contains scenario quote IDs generated by `tests/e2e/seed-booking-test-data.ts`; they are local test data, not newly invented IDs.

## Known records

- Default manual-hotspot smoke fallback: `DVI202604247` (`tests/e2e/manual-hotspot-test-utils.ts`). The helper can discover a better quote from the API when configured.
- PHP/Nest parity fixture: `DVI202604230` (`tests/e2e/itinerary-php-nest-parity.spec.ts`).
- Fit Here scenarios use configured `DVI202606167` and `DVI2026071` fixtures in their spec files; these are mutation-sensitive and must only run against the intended local test API.
- `.env.e2e` scenario records include `DVI20260485`, `DVI202604155`, `DVI202604156`, `DVI20260321`, `DVI2026024`, `DVI202604106`, and `DVI202604157` for booking-rule/hotel/guide scenarios.

## Discovery and mutation policy

The source of truth for records is the existing Playwright helper/spec and the local API response. Use read-only `GET /itineraries/details/:quoteId` during discovery. Mutation tests must use the existing reset/cleanup logic in each spec or an API-created isolated record. Never point these tests at production or document access tokens/passwords.
